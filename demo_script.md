# Nebula — 2-Minute Demo Script

**Tracks:** Developer Track + Best Use of Gemini

---

## [0:00 – 0:15] HOOK

> "Every developer has hundreds of files scattered across their machine — code, PDFs, screenshots, configs. `Cmd+F` only matches filenames. Spotlight only matches text. Neither understands what's *inside* your files."
>
> "We built **Nebula** — a desktop app that semantically indexes your entire workspace using Gemini's multimodal embeddings. Search by meaning, across every file type."

*[On screen: Nebula app open, constellation view showing project nodes floating in 3D]*

---

## [0:15 – 0:45] THE CONSTELLATION

*[Click into a project — 3D nebula of file nodes appears, colored by type]*

> "Every dot is a file in my workspace, embedded into a 768-dimensional vector with **`gemini-embedding-2-preview`**, then projected into 3D. Files that mean similar things cluster together — code with code, docs with docs, images with related images."

*[Drag camera, hover a node — tooltip shows filename]*

> "We don't just embed file bytes. PDFs get **full text extracted** with PyMuPDF. Images get **AI captions generated by `gemini-2.0-flash`** at index time — so a screenshot of a chart actually gets indexed by what's in the image, not just its filename."

---

## [0:45 – 1:20] SEMANTIC SEARCH

*[Type "resume" in the search bar]*

> "Watch — I search 'resume.'"

*[Matching nodes pulse and brighten; results panel slides open]*

> "Top hits are my actual PDF resumes — scored by cosine similarity against the query embedding in MongoDB Atlas Vector Search. No filename matching. Pure semantic."

*[Click a result → preview panel slides in showing the PDF inline]*

> "Inline preview for any file type. One click opens it in the system default app."

*[Type "golden retriever puppy" — image nodes light up]*

> "And because Gemini Flash captioned every image during indexing, I can search images by **content**. There's no metadata, no tags — Gemini just looked at the pixels and described them."

---

## [1:20 – 1:50] THE AGENT

*[Open chat panel, type: "find my resume and move it into a Resumes folder"]*

> "Nebula is also an agent. We built a LangGraph loop on **`gemini-2.0-flash`** with tool access to the same vector index and the filesystem."

*[Agent responds, plans, executes — file moves; constellation refreshes live]*

> "It ran a semantic search to locate the file, generated a filesystem plan, executed it, and the constellation **live-updated** to reflect the new state. Every action is reversible."

---

## [1:50 – 2:00] CLOSE

> "Nebula: a Tauri desktop shell, FastAPI Python backend, MongoDB Atlas Vector Search — and **Gemini at every layer**. Embeddings for the index. Vision for image understanding. Flash for the agent."

## GEMINI FLEX POINTS (when judges ask)

- **`gemini-embedding-2-preview`** — 768-dim multimodal embeddings, asymmetric `RETRIEVAL_DOCUMENT` vs `RETRIEVAL_QUERY` task types
- **`gemini-2.0-flash` vision** — captions images at index time, so they're embedded as semantic text and align cleanly in the text query space
- **`gemini-2.0-flash` agent** — LangGraph state machine with semantic search + filesystem tools, undo stack, live DB sync on rename/move
- **PyMuPDF + Gemini** — extracts PDF text before embedding so resume content scores correctly against text queries (this was a real bug we shipped a fix for)
- **MongoDB Atlas `$vectorSearch`** — sub-100ms ANN over the full index, scoped by project
